# Firebase project setup & CLI

## 1. Create the project

Firebase Console → Add project. Enable Google Analytics if you want Firebase Analytics (can't add it later without recreating).

## 2. Install Firebase CLI

Always use `npx -y firebase-tools@latest` to ensure you're running the current version — never the bare `firebase` command which may be stale or uninstalled:

```bash
npx -y firebase-tools@latest --version    # verify version
npx -y firebase-tools@latest login        # authenticate
npx -y firebase-tools@latest projects:list  # verify access
```

## 3. Register iOS and Android apps

In Firebase Console → Project settings → Your apps, register with your bundle ID / package name. Then **download config files via CLI** instead of the console — this is faster and scriptable:

```bash
# List registered apps
npx -y firebase-tools@latest apps:list --project <PROJECT_ID>

# Download Android config (outputs google-services.json)
npx -y firebase-tools@latest apps:sdkconfig ANDROID <ANDROID_APP_ID> --project <PROJECT_ID>

# Download iOS config (outputs GoogleService-Info.plist)
npx -y firebase-tools@latest apps:sdkconfig IOS <IOS_APP_ID> --project <PROJECT_ID>
```

Place both files in the **project root** (same level as `package.json`). Do not put them in `ios/` or `android/` — those are generated by prebuild and will be overwritten.

## 4. Install React Native Firebase packages

```bash
npx expo install @react-native-firebase/app
# Add only the packages you need:
npx expo install @react-native-firebase/auth
npx expo install @react-native-firebase/firestore
npx expo install @react-native-firebase/messaging
npx expo install @react-native-firebase/crashlytics
npx expo install @react-native-firebase/analytics
npx expo install @react-native-firebase/remote-config
npx expo install @react-native-firebase/storage
npx expo install expo-build-properties    # required for iOS useFrameworks config
```

Current stable: `@react-native-firebase` v21.x. All packages in the suite must be the same version.

## 5. Configure app.config.ts

```ts
// app.config.ts
export default {
  plugins: [
    [
      'expo-build-properties',
      {
        ios: {
          useFrameworks: 'static',     // REQUIRED for RNFirebase on iOS
        },
      },
    ],
    '@react-native-firebase/app',
    '@react-native-firebase/crashlytics',
    // Add others as needed; @react-native-firebase/messaging has its own plugin:
    '@react-native-firebase/messaging',
  ],
  ios: {
    googleServicesFile: './GoogleService-Info.plist',
  },
  android: {
    googleServicesFile: './google-services.json',
  },
};
```

## 6. Initialize Firebase in app entry

```ts
// app/_layout.tsx or lib/firebase.ts
import firebase from '@react-native-firebase/app';

// Firebase initializes automatically from the config files.
// Call getApp() to get the default app instance:
const app = firebase.app();
```

RNFirebase auto-initializes from the `GoogleService-Info.plist` / `google-services.json` files — you don't call `initializeApp()` manually (unlike the web SDK). Just importing `@react-native-firebase/app` is enough.

## Flutter: FlutterFire setup

```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli
# Ensure ~/.pub-cache/bin is in PATH

# Configure (auto-reads Firebase project, writes firebase_options.dart)
flutterfire configure --project=<PROJECT_ID>
```

`flutterfire configure` generates `lib/firebase_options.dart` and places `google-services.json` / `GoogleService-Info.plist` in the correct locations. Re-run whenever you add a new Firebase service or platform.

**CRITICAL**: `flutterfire configure` only handles **client config**. It does NOT provision backend services (Firestore, AI Logic, etc.). Always run `npx -y firebase-tools@latest init <service>` for backend provisioning separately.

## Firebase MCP server (Claude Code integration)

Firebase ships an MCP server inside the Firebase CLI. Connect it to Claude Code for direct project interaction — read Firestore rules, deploy functions, check Crashlytics:

```json
// .claude/settings.json
{
  "mcpServers": {
    "firebase": {
      "command": "npx",
      "args": ["-y", "firebase-tools@latest", "experimental:mcp"],
      "env": {
        "FIREBASE_TOKEN": "<your-ci-token-from-firebase-login:ci>"
      }
    }
  }
}
```

```bash
# Generate a CI token for non-interactive environments
npx -y firebase-tools@latest login:ci
```

With the Firebase MCP connected, Claude can: list Firestore databases, validate security rules, fetch Remote Config templates, tail function logs, and deploy — all without leaving the editor.

```dart
// main.dart
import 'package:firebase_core/firebase_core.dart';
import 'firebase_options.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);
  runApp(const MyApp());
}
```

pubspec.yaml dependencies:
```yaml
firebase_core: ^4.10.0
firebase_auth: ^6.5.2
cloud_firestore: ^6.5.0
firebase_messaging: ^16.3.0
firebase_crashlytics: ^5.2.3
firebase_analytics: ^11.4.2

```

## Gotchas

- **Missing `useFrameworks: 'static'`** → iOS pod install succeeds but archive fails with linker errors. Always set this when using any `@react-native-firebase` package.
- **Package version mismatch** → all `@react-native-firebase/*` packages must be on the same minor version. Mixed versions cause runtime crashes.
- **Config files in wrong location** → putting `google-services.json` inside `android/` means it gets overwritten by every `expo prebuild`. Keep them in the project root.
- **Flutter**: if `flutterfire configure` fails, ensure you have the Firebase CLI installed and are logged in (`firebase login`).
